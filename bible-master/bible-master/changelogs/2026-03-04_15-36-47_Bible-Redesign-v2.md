# Bible Redesign v2 ‚Äî Multi-Project Config, Inquiry Type, Folder Restructure

**Date:** 2026-03-04 15:36 UTC
**Scope:** All scripts, all commands, CLAUDE.md, project-config.example.json, bible-sync.md

## What Changed

### 1. Shared Config Resolver (`scripts/lib/config.js`)
- New centralized config loader replacing per-script `loadProjectConfig()` and `loadEnv()` functions
- Supports multi-project format (projects map with `--project <label>` flag) and legacy flat format
- All 14 scripts migrated: `plane-item-get.js`, `plane-item-create.js`, `plane-item-update.js`, `plane-latest-version.js`, `plane-intake-get.js`, `plane-intake-update.js`, `plane-intake-handling.js`, `plane-desc-append.js`, `plane-work-items.js`, `plane-cycle-items.js`, `outline-pull.js`, `outline-push.js`, `generate-mcp.js`, `setup-po.js`
- Exports: `loadProjectConfig(label)`, `parseProjectFlag(args)`, `buildStateMap`, `buildStateNames`, `buildEstimateMap`, `buildEstimateReverse`

### 2. Command Folder Restructure
- `.claude/commands/` split into three subfolders:
  - `shared/` ‚Äî bible-sync.md
  - `project/` ‚Äî pm.md, p.md, s.md, pp.md, rp.md, triage.md, gitpush.md, load-skills.md
  - `pa/` ‚Äî intake.md (PA-only)
- All path references updated across CLAUDE.md, docs/pm.md, docs/pm-workflow-guide.html, bible-sync.md, intake.md

### 3. INQUIRY Intake Type (`/intake` v2.0)
- `/intake` now requires explicit type prefix: `in/fe/bu/id` (inquiry/feature/bug/idea)
- INQUIRY lifecycle: create ‚Üí auto-accept ‚Üí assign Intakes module ‚Üí investigate ‚Üí Done (or spawn new intake)
- Inquiry items are self-contained ‚Äî never spawn T1/T2 directly
- Plane URL detection: scans input for existing item references, fetches context
- BUG/FEATURE/IDEA: module assignment removed from `/intake` (deferred to `/pm ACCEPT`)

### 4. Module Assignment Fix (`/pm` v2.8)
- ACCEPT and TRIAGE protocols now assign intake items to Intakes module after acceptance
- Previously done by `/intake`, but `add_work_items_to_module` fails on pending intake items
- Order: accept first ‚Üí assign module second

### 5. Multi-Project Config Format
- `project-config.json` now supports a `projects` map with `default` key
- `env` section maps canonical names to prefixed env vars (e.g., `PLANE_API_KEY` ‚Üí `WC_PLANE_API_KEY`)
- Non-secrets (URLs, slugs, UUIDs) stored in config; only API keys in `.env`
- Legacy flat format still supported for backwards compatibility

### 6. Sync Ledgers & Consumer Type Detection
- CLAUDE.md now contains two sync ledgers: Standard Consumer and PA Consumer
- `/bible-sync` auto-detects consumer type by checking for `.claude/commands/pa/` directory
- PA consumers get `intake.md` in their tracked files; standard consumers do not

### 7. Changelog System
- `changelogs/` directory for tracking Bible changes
- Format: `YYYY-MM-DD_hh-mm-ss_Title.md` (UTC)
- Rule documented in CLAUDE.md (bible-specific, not cloned to consumers)

## Why

- **Multi-project config**: The PA codebase manages multiple projects (WCL v1 on AIUR workspace, Spark on Lightcraft workspace). Scripts needed a unified way to resolve project-specific config without duplication.
- **Inquiry type**: HR and other departments send vague inquiries that don't fit BUG/FEATURE/IDEA. These need tracking but follow a different lifecycle (investigation, not implementation).
- **Folder restructure**: Commands have distinct audiences (PA vs project codebase vs shared). Folder separation makes sync cleaner and prevents PA-only commands from appearing in project codebases.
- **Module assignment fix**: Plane API silently fails when assigning modules to pending intake items. Moving assignment to after acceptance fixes this.

## Migration

### For existing project codebases (Standard Consumer)

1. **Move command files** into new subfolders:
   ```
   .claude/commands/pm.md ‚Üí .claude/commands/project/pm.md
   .claude/commands/p.md ‚Üí .claude/commands/project/p.md
   .claude/commands/s.md ‚Üí .claude/commands/project/s.md
   .claude/commands/pp.md ‚Üí .claude/commands/project/pp.md
   .claude/commands/rp.md ‚Üí .claude/commands/project/rp.md
   .claude/commands/triage.md ‚Üí .claude/commands/project/triage.md
   .claude/commands/gitpush.md ‚Üí .claude/commands/project/gitpush.md
   .claude/commands/load-skills.md ‚Üí .claude/commands/project/load-skills.md
   .claude/commands/bible-sync.md ‚Üí .claude/commands/shared/bible-sync.md
   ```

2. **Add shared config** ‚Äî copy `scripts/lib/config.js` to your project

3. **Update project-config.json** ‚Äî wrap existing flat config in multi-project format:
   ```json
   {
     "default": "myproject",
     "projects": {
       "myproject": {
         ...existing keys...,
         "env": {
           "PLANE_API_KEY": "PLANE_API_KEY",
           "OUTLINE_API_KEY": "OUTLINE_API_KEY",
           "OUTLINE_API_URL": "OUTLINE_API_URL"
         }
       }
     }
   }
   ```
   Note: legacy flat format still works, so this is optional for single-project codebases.

4. **Re-sync all scripts** ‚Äî all scripts were migrated to use `lib/config.js`. Run `/bible-sync` to pull updates.

5. **Remove dotenv dependency** ‚Äî `generate-mcp.js` no longer uses `dotenv`. You can remove it from `package.json` if no other scripts need it.

### For PA codebases

Same as above, plus:

6. **Move intake.md** ‚Üí `.claude/commands/pa/intake.md`

7. **Update `/intake` usage** ‚Äî now requires type prefix: `/intake fe|bu|in|id <description>`

8. **Multi-project config** ‚Äî if managing multiple projects, add each to `project-config.json`:
   ```json
   {
     "default": "wclv1",
     "projects": {
       "wclv1": { ... "env": { "PLANE_API_KEY": "WC_PLANE_API_KEY" } },
       "spark": { ... "env": { "PLANE_API_KEY": "LC_PLANE_API_KEY" } }
     }
   }
   ```
