# Unified Config

**Date:** 2026-03-05 12:00 UTC
**Scope:** scripts/lib/config.js, config.example.json, all scripts, SKILL.md, CLAUDE.md, bible-sync.md, /i, /intake

## What Changed

Merged `project-config.json` and `pa-config.json` into a single `config.json` with a unified `workspaces → projects` structure. Both PA and standard consumers now use the same config file and the same resolver.

### New config structure

```json
{
  "default_project": "proj-a",
  "workspaces": {
    "ws1": {
      "name": "Company A",
      "folder": "ws1",
      "comms_mcp": "ws1_ms365",
      "detection_keywords": ["teams", "outlook"],
      "projects": {
        "proj-a": {
          "PLANE_PROJECT_ID": "...",
          "env": { "PLANE_API_KEY": "WS1_PLANE_API_KEY" }
        }
      }
    }
  }
}
```

### Resolver changes

- `loadProjectConfig(label)` → `loadConfig(workspace, project)` — returns `{ workspace, project }`
- `parseProjectFlag(args)` → `parseFlags(args)` — extracts `--workspace` and `--project`
- Scripts use `loadConfig(workspace, project).project` — same downstream config shape
- PA-specific workspace fields (`folder`, `comms_mcp`, `detection_keywords`) are optional

## Why

PA and standard consumers had separate config files (`project-config.json` vs `pa-config.json`) with duplicated project constants. This caused:
- Two config resolvers to maintain
- Duplication risk when the same project constants appeared in both files
- Different adoption instructions for PA vs standard consumers

The unified config eliminates duplication. One file, one resolver, one adoption path. PA-specific fields are optional — standard consumers ignore them.

## Migration

### For standard consumers

1. Rename `project-config.json` → `config.json`
2. Wrap existing content in the new structure:
   ```json
   {
     "default_project": "myproject",
     "workspaces": {
       "default": {
         "name": "My Project",
         "projects": {
           "myproject": { /* existing project constants */ }
         }
       }
     }
   }
   ```
3. Run `/bible-sync` to update all files

### For PA consumers

1. Merge `project-config.json` and `pa-config.json` into `config.json`
2. Move project constants under their respective workspace's `projects` key
3. Delete old config files
4. Run `/bible-sync` to update all files

## Before/After

```
Before: project-config.json (flat projects) + pa-config.json (workspaces + projects)
After:  config.json (workspaces → projects, single file for all consumers)

Before: loadProjectConfig(label) + parseProjectFlag(args)
After:  loadConfig(workspace, project) + parseFlags(args)

Before: --project wclv1
After:  --project wclv1  OR  --workspace wc --project wclv1
```
